# capellaextras_deferred_index_build

Manages deferred index builds for Couchbase Capella query indexes.

When Couchbase creates an index it immediately begins building it, which requires scanning the entire
keyspace. If you are creating several indexes on the same collection you can instead create them all
with `defer_build = true` and then trigger a single build pass. This resource manages that build
step and tracks the current build status of each index.

Unlike the `capellaextras_build_index` action — which must be triggered explicitly — this resource
participates in `terraform plan`. When the stored index statuses include any value that matches
`build_trigger_statuses` (default: `["Created"]`), Terraform will detect the drift and prompt
for an apply.

## Behaviour

- **Create / Update**: Checks the current status of every index in `index_names`. Any index whose
  status matches an entry in `build_trigger_statuses` is included in a single `BUILD INDEX`
  statement. Indexes that are already building or online are left untouched.
- **Read (plan refresh)**: Fetches live statuses from the API so that drift — e.g. an index that
  was deleted and recreated as deferred outside Terraform — is surfaced on the next plan.
- **Delete**: No-op. This resource does not own the underlying indexes; destroy only removes
  the resource from Terraform state.
- **Fire-and-forget**: The build runs in the background; Terraform does not wait for indexes
  to reach `Online`. The state records `"Building"` immediately after the build is triggered
  and will be corrected to the real status on the next plan refresh.

## Extending trigger statuses

By default only indexes in `"Created"` status are built. You can extend this via
`build_trigger_statuses` to handle additional states, for example to rebuild indexes that have
entered an error state:

```hcl
resource "capellaextras_deferred_index_build" "indexes" {
  # ...
  build_trigger_statuses = ["Created", "Error"]
}
```

## Example Usage

```terraform
locals {
  org_id = "aaaaaaaa-8f0c-22222-865e-bbbbbbbbbbbb"
  indexes = {
    idx1 = { index_keys = ["foo", "bar"] }
    idx2 = { index_keys = ["baz"] }
    idx3 = { index_keys = ["qux", "quux"] }
  }
}

# Create all indexes with defer_build = true so the keyspace is only scanned once.
resource "couchbase-capella_query_indexes" "index" {
  for_each        = local.indexes
  organization_id = local.org_id
  project_id      = couchbase-capella_project.new_project.id
  cluster_id      = couchbase-capella_free_tier_cluster.new_free_tier_cluster.id
  bucket_name     = couchbase-capella_bucket.new_free_tier_bucket.name
  index_name      = each.key
  index_keys      = each.value.index_keys
  with = {
    defer_build = true
  }
}

# This resource tracks whether each index has been built and triggers a build for
# any index still in the "Created" state. Re-running terraform apply after an index
# is deleted and recreated (deferred) will automatically re-trigger the build.
resource "capellaextras_deferred_index_build" "indexes" {
  organization_id = local.org_id
  project_id      = couchbase-capella_project.new_project.id
  cluster_id      = couchbase-capella_free_tier_cluster.new_free_tier_cluster.id
  bucket_name     = couchbase-capella_bucket.new_free_tier_bucket.name

  index_names = [for idx in couchbase-capella_query_indexes.index : idx.index_name]

  depends_on = [couchbase-capella_query_indexes.index]
}
```

## Schema

### Required

- `bucket_name` (String) The bucket where the indexes are located.
- `cluster_id` (String) The cluster ID where the indexes are located.
- `index_names` (List of String) The names of the deferred indexes to manage builds for.
- `organization_id` (String) The organization ID where the indexes are located.
- `project_id` (String) The project ID where the indexes are located.

### Optional

- `build_trigger_statuses` (List of String) Index statuses that should trigger a deferred build. Defaults to `["Created"]`. Extend this list to include additional statuses (e.g. error states) that should also trigger a rebuild.
- `collection_name` (String) The collection where the indexes are located. Defaults to `_default`.
- `scope_name` (String) The scope where the indexes are located. Defaults to `_default`.

### Read-Only

- `id` (String) Composite identifier: `{organization_id}/{project_id}/{cluster_id}/{bucket_name}`.
- `index_statuses` (Map of String) Current build status of each managed index, keyed by index name. Updated after each apply and refreshed on `terraform plan`.
